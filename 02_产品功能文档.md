# 产品功能文档（当前最终整合版）

## 已具备的核心功能

### 1. 文本问答（主能力）
- 按产品知识库进行问答
- 支持 `brief / full` 两种回答模式
- 优先走规则提取（章节抽取），不足时再走检索结果整理
- 适合固定资料型问题：
  - 基础资料
  - 操作说明
  - 术后护理
  - 禁忌人群
  - 防伪鉴别

### 2. 索引构建
- 从 `knowledge/<product>/main.txt / faq.txt / alias.txt` 建索引
- 自动分块
- 生成 `stores/<product>/index.faiss` 与 `docs.jsonl`
- 支持按产品单独重建：`--product feiluoao`

### 3. 查询增强
- Query Rewrite（问题扩写/同义词补充）
- alias 别名召回
- brief/full 输出模板控制
- 未命中时记录日志

### 4. 媒体关联
- 根据问题类型或答案内容返回相关图片/视频
- 使用 `knowledge/<product>/media.json` 建立映射
- 不影响主检索结果，只是附加展示

### 5. 后台管理
- 列出已存在产品
- 触发重建索引
- 查看 QA 日志
- 查看未命中日志
- 查看错误日志
- 提供一个简易后台 HTML 页

## 更适合当前版本的使用场景

- 医美门店内部知识问答
- 咨询师快速查固定产品资料
- 培训型资料检索
- 标准化回答（非开放医学诊断）

## 当前版本的边界

### 能做得好
- 针对**已录入知识库**的固定资料问答
- 针对**单一产品**的结构化检索
- 给用户返回相关媒体（图/视频）

### 仍需注意
- 对完全不在知识库里的开放性医美问题，回答质量取决于兜底模板
- 多产品、多项目组合问法虽然有增强，但仍建议继续优化拆分策略
- 不是诊断系统，仍应保留“仅供参考，需专业医师评估”提示

## 最推荐的知识库组织方式

每个产品至少 4 份文件：
- `main.txt`：完整主资料
- `faq.txt`：高频问答
- `alias.txt`：别名、同义词、检索词
- `media.json`：图片/视频映射（可选）
