# 所有接口详细作用（基于当前最终整合版）

## 一、主问答 API（api_server.py）

### 1) GET `/health`
作用：检查服务是否正常启动。

典型返回：
- `status`
- `rag_script_exists`
- `answer_file_exists`
- `python_exe`
- `base_dir`

### 2) POST `/ask`
作用：提交文本问题并返回问答结果。

请求字段：
- `question`：用户问题
- `k`：可选，手动指定检索条数
- `mode`：`brief` / `full`
- `timeout_sec`：子进程超时时间
- `debug`：是否返回调试信息

返回字段：
- `ok`
- `question`
- `mode`
- `answer`
- `source`
- `return_code`
- `stderr`
- `debug`

用途：这是主业务接口，前端对话页一般调用它。

---

## 二、媒体增强 API（media/api_server_media.py）

### 1) GET `/health`
作用：媒体 API 的健康检查。

### 2) GET `/chat`
作用：返回一个简易网页，可直接在浏览器里测试问题输入、查看答案与媒体。

### 3) POST `/ask_media`
作用：返回“答案 + 关联图片/视频”。

请求字段：
- `question`
- `product`
- `k`
- `mode`
- `timeout_sec`
- `with_media`
- `max_media`
- `debug`

返回字段：
- `ok`
- `question`
- `product`
- `mode`
- `qtype`
- `answer`
- `media`（数组）
- `source`
- `return_code`
- `stderr`
- `debug`

`media` 单项常见字段：
- `title`
- `type`（image / video）
- `url`
- `source`
- `note`

用途：适合门店/咨询前台展示“文字 + 图片/视频”。

---

## 三、后台管理 API（admin/admin_api.py）

### 1) GET `/`
作用：直接打开后台管理 HTML 页面（`admin_page.html`）。

### 2) GET `/admin/health`
作用：后台健康检查。

### 3) GET `/admin/products`
作用：列出 `knowledge/` 下所有产品目录，以及各自包含的文件。

### 4) POST `/admin/rebuild`
作用：根据指定产品重建索引。

请求字段：
- `product`
- `timeout_sec`

返回字段：
- `ok`
- `cmd`
- `return_code`
- `stdout`
- `stderr`

### 5) GET `/admin/logs/qa`
作用：查看最近问答日志。

### 6) GET `/admin/logs/miss`
作用：查看最近未命中日志。

### 7) GET `/admin/logs/error`
作用：查看最近错误日志。

用途：适合你自己做运维、排查知识库命中和索引问题。
