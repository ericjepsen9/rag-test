# 当前版本完整框架结构（基于最终整合包）

> 基于：`rag_final_complete_with_admin_knowledge_and_media_20260228.zip`

## 顶层目录

```text
final_complete/
├─ rag_answer.py                  # 主问答引擎（检索、规则路由、回答生成）
├─ build_faiss.py                 # 向量索引构建脚本
├─ rag_runtime_config.py          # 运行配置（开关、阈值、模式、日志等）
├─ api_server.py                  # 主对话 API（文本问答）
├─ query_rewrite.py               # 问题改写器（查询扩展/重写）
├─ answer_formatter.py            # 结构化答案格式化
├─ media_router.py                # 图片/视频关联路由
├─ rag_media_config.py            # 媒体映射配置
├─ run_regression.py              # 回归测试脚本
├─ rag_logger.py                  # 日志记录（QA/未命中/错误）
├─ README_FINAL.md                # 主说明文档
│
├─ knowledge/                     # 知识库目录（按产品分文件夹）
│  └─ feiluoao/
│     ├─ main.txt                 # 主资料（完整正文）
│     ├─ faq.txt                  # FAQ 问答资料
│     ├─ alias.txt                # 别名/检索词
│     └─ media.json               # 图片/视频映射（可选）
│
├─ stores/                        # 构建后的向量索引输出目录
│  └─ <product>/
│     ├─ index.faiss              # FAISS 索引
│     └─ docs.jsonl               # 分块后的文本与元数据
│
├─ media/                         # 媒体增强模块
│  ├─ api_server_media.py         # 带媒体返回的 API
│  └─ README_MEDIA_GUIDE.md       # 媒体功能说明
│
└─ admin/                         # 后台管理模块
   ├─ admin_api.py                # 后台 API
   ├─ admin_page.html             # 后台管理页
   └─ README_ADMIN_ENHANCEMENT.md # 后台说明
```

## 运行关系

1. `knowledge/<product>/main.txt / faq.txt / alias.txt` 提供原始资料。
2. `build_faiss.py` 读取知识库并切块，输出到 `stores/<product>/`。
3. `rag_answer.py` 读取 `stores/<product>/index.faiss + docs.jsonl` 做检索与回答。
4. `api_server.py` 把 `rag_answer.py` 包装成 HTTP 接口。
5. `api_server_media.py` 在主答案基础上叠加图片/视频推荐。
6. `admin_api.py` 用于重建索引、查看产品列表、查看日志。

## 推荐部署方式

- **基础版**：`rag_answer.py + build_faiss.py + knowledge + stores`
- **API版**：基础版 + `api_server.py`
- **媒体版**：API版 + `media_router.py + rag_media_config.py + media/`
- **后台版**：媒体版 + `admin/ + rag_logger.py`
