# RAG 医美问答系统 — 产品逻辑、功能与准确性评估

> 评估日期：2026-03-17

---

## 一、系统架构总览

```
用户提问
  ↓
┌─────────────────────────────────┐
│  1. 闲聊检测 (你好/谢谢/再见)    │ → 直接返回固定回复
│  2. 特殊意图检测 (价格/对比/地点) │ → 返回模板兜底回复
└─────────────────────────────────┘
  ↓ （非闲聊/非特殊意图）
┌─────────────────────────────────┐
│  3. 查询预处理                    │
│   - 产品识别 (菲罗奥/赛罗菲/别名) │
│   - 项目识别 (水光/微针/光电)      │
│   - 路由检测 (18+路由)            │
│   - 查询改写 (同义词/LLM扩写)     │
│   - 子问题拆分 (复合问题≤4个)      │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│  4. 多阶段检索                    │
│   ① 章节定位 (SECTION_RULES)      │ → 匹配到直接返回（最快）
│   ② FAQ 快速路径匹配              │ → FAQ精准命中直接返回
│   ③ FAISS 向量搜索 (BGE-M3)      │
│   ④ BM25 关键词搜索 (jieba分词)   │
│   ⑤ 混合排序 (65%向量+35%关键词)  │
│   ⑥ 动态阈值过滤                  │
│   ⑦ [可选] BGE-M3 Reranking      │
└─────────────────────────────────┘
  ↓
┌─────────────────────────────────┐
│  5. 答案生成                      │
│   - 证据合并 (最多6个chunk)        │
│   - 结构化格式 (brief/full)       │
│   - 风险/禁忌声明自动附加          │
│   - [可选] LLM润色生成            │
│   - 媒体关联 (图片/视频)          │
└─────────────────────────────────┘
  ↓
返回 JSON（含答案、来源、路由、延迟、调试信息）
```

## 二、核心功能模块详解

### 2.1 问题路由系统 (18+路由)

| 路由 | 覆盖场景 | 关键词示例 |
|------|---------|-----------|
| `risk` | 不良反应/副作用 | 红肿、结节、感染、淤青 |
| `aftercare` | 术后护理 | 洗脸、面膜、运动、化妆、防晒 |
| `operation` | 操作参数 | 注射深度、针头、麻醉 |
| `anti_fake` | 防伪鉴别 | HiddenTag、正品、真伪 |
| `contraindication` | 禁忌人群 | 孕妇、过敏、18岁以下 |
| `ingredient` | 成分解析 | PCL、透明质酸、生长因子 |
| `basic` | 产品基础 | 是什么、备案、规格 |
| `effect` | 效果/维持 | 维持多久、见效、变白 |
| `combo` | 联合方案 | 一起做、搭配、间隔 |
| `pre_care` | 术前准备 | 术前检查、停药、素颜 |
| `design` | 方案设计 | 怎么打、几支、松弛程度 |
| `repair` | 修复补救 | 不理想、不均匀、做坏了 |
| `complication` | 并发症处理 | 术后第X天、正常吗 |
| `course` | 疗程规划 | 几次、间隔多久、时间表 |
| `anatomy_q` | 部位咨询 | 苹果肌、法令纹、颈部 |
| `indication_q` | 适应症推荐 | 松弛怎么改善、毛孔粗大 |
| `procedure_q` | 项目咨询 | 水光针是什么、项目对比 |
| `equipment_q` | 设备咨询 | 什么仪器、水光仪参数 |
| `script` | 话术指导 | 客户担心、怎么介绍 |

**消歧机制**：
- 路由优先级排序 (`_ROUTE_ORDER`)：complication > script > procedure_q > equipment_q > anatomy_q > indication_q > course > risk > repair > combo > aftercare > ...
- 时间线消歧：短期(术后第2天) → complication，长期(术后3个月) → risk
- 上下文信号词：如"修复"关键词将 risk 重定向到 repair
- 多路由交叉时优先匹配更具体的路由

### 2.2 知识库架构

```
knowledge/
├── feiluoao/          ← 产品专属知识 (main.txt + faq.txt + alias.txt)
├── procedures/        ← 共享项目知识 (filling/microneedling/photoelectric/water_light)
├── equipment/         ← 共享设备知识 (mts_device/rf_device/water_light_machine)
├── materials/         ← 共享材料知识 (hyaluronic_acid/collagen)
├── anatomy/           ← 解剖与部位 (面部8区域+身体2区域)
├── complications/     ← 并发症处理 (时间线+症状决策树)
├── courses/           ← 疗程规划 (3套方案+3套联合时间表)
├── indications/       ← 适应症推荐 (6种皮肤问题+4个年龄段)
└── scripts/           ← 话术指导 (8个常见客户问题+合规表达)
```

### 2.3 检索策略

| 策略 | 权重/优先级 | 适用场景 |
|------|-----------|---------|
| 章节定位 | 最高优先 | 防伪、术后护理、禁忌等有固定章节的问题 |
| FAQ快速路径 | 次高优先 | 精确匹配FAQ问答对 |
| 向量搜索 (FAISS) | 65% | 语义相似度高的场景 |
| 关键词搜索 (BM25) | 35% | 精确术语匹配 |
| 动态阈值 | 过滤层 | 自适应过滤低质量结果 |
| Reranking | 可选 | BGE-M3 多维度精排 |

## 三、能否实现快速、准确的输出？

### 3.1 速度评估

| 环节 | 预估耗时 (GPU) | 预估耗时 (CPU) | 优化措施 |
|------|--------------|--------------|---------|
| 查询预处理 | <10ms | <10ms | 预编译正则、frozenset查找 |
| 路由检测 | <5ms | <5ms | 关键词预计算为小写 |
| 章节定位 | <1ms | <1ms | 直接字符串匹配 |
| FAQ快速路径 | <50ms | <200ms | 向量编码+余弦相似度 |
| 向量搜索 | ~50ms | ~500ms | FAISS HNSW + 缓存 |
| BM25搜索 | ~20ms | ~20ms | jieba分词+预构建索引 |
| 答案组装 | <10ms | <10ms | 模板拼接 |
| LLM生成(可选) | 500-2000ms | 同左 | 需网络调用，可关闭 |
| **总计(无LLM)** | **~150ms** | **~750ms** | |
| **总计(含LLM)** | **~1-3s** | **~1-3s** | |

**速度结论**：
- **GPU环境下，无LLM模式**：响应 <200ms，可实现近实时问答
- **CPU环境下，无LLM模式**：响应 ~1s 内，用户体验尚可
- **含LLM润色**：受API延迟影响，通常 1-3s
- 线程池并行搜索（6 workers）显著降低了检索总耗时
- 多级缓存（embedding、FAISS索引、查询改写）有效减少重复计算

### 3.2 准确性评估

**优势**：
1. **多阶段检索** — 章节定位 > FAQ匹配 > 向量+关键词混合，逐级兜底
2. **精细路由** — 18+路由覆盖医美场景，消歧机制处理歧义查询
3. **领域词典** — jieba自定义词典包含大量医美术语，分词准确
4. **别名覆盖** — 产品名（菲罗奥/赛罗菲/非罗奥/CELLOFILL等）、项目名、设备名均有别名映射
5. **动态阈值** — 自适应过滤低质量结果，避免返回无关内容
6. **安全声明** — 风险/禁忌类自动附加免责声明

**潜在风险点**：
1. **知识库覆盖有限** — 当前仅有菲罗奥一个产品，无法回答其他品牌产品问题
2. **口语化查询** — 虽有消歧机制，但极端口语化（如"脸垮了"、"出事了"）可能路由偏差
3. **复合问题** — 子问题拆分最多4个，超复杂问题可能丢失信息
4. **无LLM时答案质量** — 纯检索模式下答案为原文段落截取，可读性不如LLM润色
5. **时效性** — 知识库为静态文件，产品更新需手动维护

### 3.3 综合结论

| 维度 | 评分 | 说明 |
|------|------|------|
| 响应速度 | ★★★★☆ | GPU下<200ms，CPU下<1s，含LLM约1-3s |
| 答案准确性 | ★★★★☆ | 知识库范围内准确率高，消歧机制成熟 |
| 知识覆盖度 | ★★★☆☆ | 菲罗奥产品详尽，但仅单产品，共享知识覆盖基本医美项目 |
| 边界处理 | ★★★★☆ | 价格/地点/闲聊等边界场景有兜底 |
| 安全合规 | ★★★★★ | 风险声明、禁忌提醒、专业医师引导完善 |
| 可扩展性 | ★★★★☆ | 多产品架构，新产品只需添加 knowledge/<product>/ |

**总体评价**：该系统在当前知识库范围内可以实现**快速且准确**的医美问答输出。主要限制在于知识库覆盖范围（仅菲罗奥一个产品），而非系统能力本身。

## 四、建议改进方向

1. **扩充产品线** — 添加更多医美产品知识库（如其他品牌水光、肉毒素、玻尿酸填充剂等）
2. **对话上下文** — 增强多轮对话能力，记住用户前序问题
3. **答案引用** — 在答案中标注具体来源章节，增强可信度
4. **监控仪表板** — 现有miss日志可进一步聚合分析，发现知识库盲区
5. **A/B测试** — LLM vs 纯检索模式的用户满意度对比
