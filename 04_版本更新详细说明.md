# 版本更新详细说明（整理版）

## 第一阶段：基础 RAG 版
- 建立 `main / faq / alias` 三文件知识库结构
- 通过 `build_faiss.py` 构建 FAISS 索引
- `rag_answer.py` 通过本地脚本回答并写入 `answer.txt`

**解决的问题**：先把固定产品资料查出来。

---

## 第二阶段：规则优先稳定化
- 加入按问题类型的规则回答（防伪 / 禁忌 / 术后 / 操作）
- 避免完全依赖相似度拼接，减少串段
- 增加 `brief / full` 模式

**提升点**：回答更像“从资料里提取”，不是胡乱拼接。

---

## 第三阶段：配置化与多产品方向
- 抽出运行配置到 `rag_runtime_config.py`
- 增加 query rewrite / alias 扩写
- 为多产品结构预留目录方案：`knowledge/<product>`、`stores/<product>`

**提升点**：不同脚本不再各自写死参数，维护成本下降。

---

## 第四阶段：API 化
- 增加 `api_server.py`
- 支持通过 HTTP 调用问答
- 便于网页或其他客户端接入

**提升点**：从命令行脚本升级为可对外服务。

---

## 第五阶段：媒体增强
- 增加 `media_router.py`
- 增加 `media/api_server_media.py`
- 支持 `media.json` 做图片/视频映射
- 返回答案时同时返回相关媒体

**提升点**：更适合实际展示，不只是纯文字。

---

## 第六阶段：后台管理与日志
- 增加 `admin/admin_api.py`
- 增加 `admin_page.html`
- 增加 `rag_logger.py`
- 可查看 QA / miss / error 日志
- 可在后台触发重建索引

**提升点**：开始具备“可维护、可运营”的后台能力。

---

## 第七阶段：整合最终包
- 将主问答、媒体、后台、knowledge、stores 打成一个整包
- 输出 `README_COMBINED.md / README_FINAL.md / PRODUCTION_DEPLOY_GUIDE`
- 形成“最终生产整合版”交付结构

**当前结论**：
这个版本已经是一个**能跑、能维护、可扩展**的本地医美知识库系统；真正再上一个台阶，就是继续增强：
- 多实体拆分
- 时间轴知识库
- 联合方案知识库
- 更强的后台录入能力
- 更自动化的媒体映射生成
